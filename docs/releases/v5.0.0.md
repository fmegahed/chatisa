# ChatISA v5.0.0 Release Notes

**Release Date:** December 18, 2025

---

## Overview

Version 5.0.0 brings significant UI/UX improvements with consistent sidebar design across all modules, enhanced AI Sandbox functionality, and important bug fixes for model compatibility.

### State-of-the-Art Models

This release includes the latest SOTA models from leading AI providers:
- **OpenAI**: GPT-5.2 and GPT-5 Mini (December 2025) - featuring enhanced reasoning and vision capabilities
- **Anthropic**: Claude Sonnet 4.5 (September 2025) - exceptional code generation and analysis
- **Google**: Gemini 3 Pro Preview and Gemini 3 Flash Preview (November 2025) - advanced reasoning with extended thinking

ChatISA now supports **17 models across 6 providers**, ensuring students have access to cutting-edge AI technology for their educational needs.

## Bug Fixes

### Model Compatibility
- **Gemini 3 Pro Preview**: Fixed empty response issue caused by reasoning tokens consuming the entire output limit. Now uses model's configured `max_tokens` from config
- **GPT-5 Mini Temperature**: Fixed "Unsupported parameter: 'temperature'" error by conditionally excluding temperature for models that don't support it
- **AI Sandbox Image Download**: Fixed `'HttpxBinaryResponseContent' object is not subscriptable` error by using direct HTTP requests for container file downloads

### UI Fixes
- Fixed broken emoji icons (showing as "?") in AI Comparisons by replacing with text labels like `[Vision]`, `[Open]`, `[Free]`

## Configuration Changes

### New Model Tag
Added `"sandbox"` tag to model configuration for AI Sandbox compatibility:
```python
# config.py - Models with sandbox tag
"gpt-5.2-2025-12-11": {
    "tags": ["premium", "coding", "reasoning", "large_context", "transcription", "sandbox"]
},
"gpt-5-mini-2025-08-07": {
    "tags": ["fast", "cost_effective", "coding", "transcription", "sandbox"]
}
```

To add new models to AI Sandbox, simply add `"sandbox"` to their tags array.

### Sidebar Configuration
- `render_sidebar()` now accepts `skip_navigation` parameter for custom page layouts
- New `render_clear_conversation_button()` function for consistent clear functionality

## Upgrade Notes

This release is backward compatible. No configuration changes are required for existing deployments.

To take advantage of the new `"sandbox"` tag system for AI Sandbox model selection, ensure your `config.py` includes the `"sandbox"` tag for any models you want available in the AI Sandbox module.
